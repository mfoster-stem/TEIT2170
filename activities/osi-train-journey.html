Common ports reference (Transport layer)
PortProtocolWhat it's for

The destination port is how a receiving computer, after every other header is stripped away, knows exactly which application should get the data. The source port is how the reply finds its way back.

Note on layers 5–7: OSI treats Session, Presentation, and Application as three separate stops, each conceptually adding its own control info. In real TCP/IP traffic that work usually happens inside the application protocol itself (TLS handles encryption and session state, for instance) rather than as three distinct headers — this line keeps them as separate stations because that's the job split OSI defines, not because a packet capture would show three headers.
